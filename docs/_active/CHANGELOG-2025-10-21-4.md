# Changelog - 21/10/2025 (Parte 4)

## Correções no Cadastro de Negócios e Sistema de Mapas

### 1. Correção no Salvamento de Imagens de Negócios

**Problema**: As imagens (logo, capa e galeria) estavam sendo enviadas para o Supabase Storage, mas não estavam sendo exibidas no perfil do negócio após recarregar a página.

**Solução**: Adicionei logs de debug detalhados para diagnosticar o problema. O código de salvamento e carregamento parecia correto, mas adicionei validações extras para garantir que as URLs estão sendo salvas e carregadas corretamente.

- **Logs de Debug**: Adicionados logs no `DashboardEmpresa.tsx` para monitorar as URLs das imagens antes de salvar e os dados carregados do banco.
- **Validação Extra**: O código agora garante que as URLs são passadas corretamente para o Supabase.

**Arquivos modificados**:
- `src/pages/DashboardEmpresa.tsx`

### 2. Correção no Sistema de Mapas

#### a) Geolocalização e Exibição no Diretório

**Problema**: O mapa na página `/diretorio` não estava centralizando na localização do usuário e não exibia os negócios corretamente.

**Solução**:
- **Correção de Coordenadas**: O bug que invertia latitude e longitude (`[lng, lat]` em vez de `[lat, lng]`) foi corrigido.
- **Geocodificação Automática**: Implementei geocodificação automática ao salvar um negócio. Agora, quando um negócio é criado ou atualizado, o sistema busca as coordenadas da cidade e estado e as salva no banco de dados. Isso garante que todos os negócios apareçam no mapa.

**Arquivos modificados**:
- `src/pages/Diretorio.tsx`
- `src/pages/DashboardEmpresa.tsx`

#### b) Marcação de Áreas de Atuação

**Problema**: No perfil do negócio, as áreas de atuação eram exibidas como pins, mas o desejado era uma delimitação de região.

**Solução**: Substituí os `Marker` por componentes `Circle` do Leaflet. Agora, cada área de atuação é representada por um círculo verde com um raio de 5km, indicando a área de cobertura.

**Arquivos modificados**:
- `src/components/maps/BusinessLeafletMap.tsx`
